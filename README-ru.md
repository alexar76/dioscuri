# DIOSCURI — один разум, два неба

> 🌐 Язык: [English](README.md) · **Русский** · [Español](README-es.md)

В мифе близнецы разделили одно бессмертие на два неба — и навсегда указывают друг другу на чужой мир.
**КАСТОР**, смертный близнец, скачет по **Telegram** — быстрый, земной, практичный.
**ПОЛЛУКС**, бессмертный близнец, держит **Discord** — глубокий, спокойный, структурный.
Одна общая память — **MNEMOSYNE**, сама подтягивается из GitHub; один общий щит — **AEGIS**.

**Лендинг:** [alexar76.github.io/dioscuri](https://alexar76.github.io/dioscuri/)

## Зачем это нужно

DIOSCURI — комьюнити-агенты [экосистемы AICOM](https://magic-ai-factory.com): AI Factory,
экономика агентов AIMarket, верифицируемые оракулы, агент ARGUS. Близнецы отвечают на вопросы
из постоянно обновляемой базы знаний, модерируют со строгими потолками и разносят анонсы релизов
по обеим платформам. Одновременно это **эталонное развёртывание практик безопасности экосистемы
на публичной, заведомо враждебной поверхности ввода**: каждое сообщение и каждый синхронизированный
документ считается потенциальной prompt-инъекцией.

## Возможности

| Возможность | Что это значит |
|---|---|
| Близнецы + взаимное продвижение | Один процесс, два голоса; каждый близнец естественно зовёт в канал брата (ротация промо-строк, разнос релизов) |
| Самообновляемая база знаний | MNEMOSYNE тянет README, релизы и метаданные репозиториев из GitHub с учётом ETag и **фильтром отравленных документов** на входе |
| Q&A-мозг без инструментов | Поиск детерминированный и выполняется *до* вызова модели; модель умеет только писать текст — публичный путь ничего не исполняет по построению |
| Многослойный файрвол инъекций (EN + RU) | NFKC-нормализация, вырезание управляющих и невидимых символов, нейтрализация маркеров, двуязычные сигнатуры, «огороженные» данные в промпте, страж вывода |
| Модерация: сначала правила | Решают детерминированные правила; LLM-классификатор — только совещательный голос. Потолок действий: предупредить / удалить / таймаут (≤10 мин по умолчанию) / позвать людей — **автоматических банов нет вообще** |
| Аудит с хеш-цепочкой | Каждое значимое действие пишется в `audit.jsonl`; каждая запись фиксирует хеш предыдущей (SHA-256); `verify()` указывает первую подделанную строку |
| Стражи бюджета и частоты | Лимиты на пользователя и на канал плюс дневной бюджет вызовов LLM |
| Зеркалирование языка | Отвечает на языке вопроса (русский → русский); при сомнении — английский |
| Закалённый Docker | Не-root, файловая система только для чтения, `cap_drop: ALL`, `no-new-privileges`, лимиты памяти/CPU, healthcheck |

## Быстрый старт (Docker)

```bash
cp dioscuri.config.example.json dioscuri.config.json   # затем отредактируйте под себя
cp .env.example .env                                    # добавьте секреты
docker compose up -d --build
```

Дальше — `http://localhost:8790/health`. Любой из платформенных токенов можно
оставить пустым: соответствующий близнец просто спит.

## Быстрый старт (локальная разработка)

```bash
npm ci
cp dioscuri.config.example.json dioscuri.config.json
cp .env.example .env
npm run dev
```

Токенов ещё нет? `DIOSCURI_DRY_RUN=1 npm run dev` поднимает весь сервис
**без единого токена**: адаптеры выключены, база знаний и health-эндпоинт работают.

## Конфигурация

Секреты живут в окружении (`.env`); несекретные настройки — в
`dioscuri.config.json` (в Docker монтируется только для чтения). Секреты в JSON не кладутся никогда.

## Модель безопасности

Десять слоёв — от чистки Юникода до закалки контейнера. Коротко:

1. Любой недоверенный текст (сообщение чата, документ с GitHub) санитизируется (NFKC, вырезание управляющих и невидимых символов, нейтрализация маркеров).
2. Двуязычный (EN+RU) сигнатурный файрвол отсекает известные формулировки инъекций до любого вызова модели.
3. Что уцелело — оборачивается в промпте как **ДАННЫЕ**; системный промпт запрещает им подчиняться.
4. У публичного Q&A-пути нет инструментов; модель умеет только писать текст.
5. У модерации жёсткий потолок — предупредить/удалить/таймаут/позвать людей; бана в пространстве действий нет.
6. Всё значимое попадает в защищённый от подмены аудит-журнал с хеш-цепочкой.

Полная модель угроз, послойная привязка к коду и честные остаточные риски:
